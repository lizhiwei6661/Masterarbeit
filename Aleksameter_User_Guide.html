1. What is Aleksameter?

Aleksameter is a desktop application for analyzing spectral reflectance measurements and calculating color coordinates. It takes your CSV measurement files and converts them into meaningful color data that you can use for research, quality control, or color matching.

What does it do?

üìà Processes Spectral Data

Reads CSV files containing wavelength and reflectance measurements (380-780nm range)

üé® Calculates Colors

Converts reflectance to CIE x,y coordinates and sRGB color values

üìä Shows Visual Results

Displays spectral curves, CIE color diagram, and data tables

üíæ Exports Results

Saves data to Excel, CSV, or high-quality plot images

4. Importing Your Data

File Format Requirements

Aleksameter reads CSV files only with this format:

Wavelength [nm], Sample1, Sample2, Sample3, ... 380, 0.123456, 0.234567, 0.345678, ... 385, 0.125678, 0.236789, 0.347890, ... 390, 0.127890, 0.238901, 0.349012, ... ... 780, 0.543210, 0.654321, 0.765432, ...
‚ö†Ô∏è Important:
  • First column must be wavelengths in nanometers
  • Use decimal points (not commas) for numbers
  • Header row must include "Wavelength [nm]"
  • Reflectance values should be 0-1 (not percentages)

Two Import Modes

General Mode (Most Common)

Use this for data that's already calibrated or from reference databases.

1 Click File ‚Üí Import or press Ctrl+O
2 Select "General" mode
3 Browse and select your CSV files (can select multiple)
4 Preview appears showing up to 3 files
5 Click OK to import

Aleksameter Mode (With Calibration)

Use this when you have raw instrument data that needs black/white reference calibration.

1 Select "Aleksameter" mode
2 Import Black Reference file first
3 Import White Reference file
4 Import your measurement files
5 Software applies calibration formula automatically
üí° Batch Processing: You can select multiple measurement files at once using Ctrl+Click (Windows/Linux) or Cmd+Click (Mac)

8. Common Issues & Solutions

Import Problems

Problem: "File format not recognized"
Solution: Check that your file is CSV format with "Wavelength [nm]" in the first column header
Problem: Data looks wrong after import
Solution: Verify decimal points (not commas) and that reflectance values are 0-1, not percentages

Calculation Issues

Problem: Negative reflectance values
Solution: Check black reference calibration, ensure proper file order in Aleksameter mode
Problem: Colors outside CIE boundary
Solution: Verify measurement accuracy, check for data corruption or calibration errors

Display Problems

Problem: Charts appear blank
Solution: Try Edit ‚Üí Clear and re-import data, check that files contain valid numbers

9. Frequently Asked Questions

Q: What file formats does Aleksameter support?

A: Only CSV files for import. Export supports Excel, CSV, TXT, and JSON formats.

Q: What wavelength range should my data cover?

A: The visible spectrum from 380-780 nanometers. Data can be at 1nm or 5nm intervals.

Q: When should I use Aleksameter mode vs General mode?

A: Use Aleksameter mode when you have raw instrument data that needs black/white reference calibration. Use General mode for pre-calibrated data.

Q: What is the rho_lambda value?

A: It's a calibration correction factor (default 0.989) that accounts for instrument-specific characteristics. Only adjust if you know your instrument's specific value.

Q: Can I process multiple samples at once?

A: Yes! Select multiple CSV files during import. Each column (after wavelength) is treated as a separate sample.

Q: Why are my RGB values different from what I expected?

A: Color appearance depends on the illuminant and observer. Try changing the illuminant in settings (D65 is most common).

Q: How do I save my work?

A: Use File ‚Üí Export to save your calculated results. Settings are automatically saved between sessions.
